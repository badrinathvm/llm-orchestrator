Context

README.md already lists this as pattern 3 — "reason → act → observe, looping until the agent decides it's done" — with the endpoint marked — and the write-up "coming next." The repo has two graphs already: POST /query (adaptive routing) and POST /orchestrate (orchestrator-workers), both hand-rolled LangGraph StateGraphs, not langgraph.prebuilt.create_react_agent. This plan follows that same convention — explicit reason and act nodes you can read top to bottom — and it's the first time the repo does any tool-calling at all: a repo-wide grep for bind_tools / @tool / ToolNode currently returns nothing.

Use case: an order-support operations agent. A customer asks something like "my order arrived damaged, can I get a refund?" — the agent loops through Reason → Act → Observe, pulling in whatever it needs, until it can give a final answer or hand off to a person. This lines up with the source diagram's own framing: "Best for: workflow automation, operations agents."

02

The loop

Two graph nodes, one conditional edge, one safety valve.

reason → act → observe, looping back to reason until iterations run out or the model answers directly

reason(state)

Binds the four tools and invokes the LLM over the running message list. If it comes back with tool_calls, the loop continues; if not, its content becomes the final answer. Past max_iterations, the LLM is invoked without tools bound — forcing a direct answer instead of another loop, the safety valve against runaway autonomy.

act(state)

Dispatches each tool call by name, wraps the result as a ToolMessage, and appends one trace entry per call: { iteration, thought, action, action_input, observation }.

server.py — new handler

server.py
@app.post("/agent", response_model=AgentResponse)
async def agent(req: QueryRequest):
    if not req.question.strip():
        raise HTTPException(status_code=400, detail="question must not be empty")

    response = await asyncio.to_thread(ReactGraphBuilder().run, req.question)
    tools_used = sorted({s["action"] for s in response["trace"]})

    return AgentResponse(
        answer=response["answer"],
        iterations=response["iterations"],
        tools_used=tools_used,
        trace=response["trace"],
    )

Verification

  1. Install the new dependency — pip install -r requirements.txt (or uv pip install -r requirements.txt) to pick up langchain-tavily.
  2. Add TAVILY_API_KEY to .env (free key from Tavily).
  3. Run python server.py, then send: curl -X POST localhost:8000/agent -d '{"question": "My order A1002 arrived damaged, can I get a refund?"}' — confirm a non-empty answer, a multi-step trace (lookup_order → search_policy → calculate_refund), and a small iterations count.
  4. Try a second question that should trigger escalate_to_human — an unknown order id, or something outside policy — and confirm the agent branches to it instead of guessing.
  5. Temporarily set max_iterations=1 and confirm the agent still returns a forced final answer rather than erroring or looping forever.
