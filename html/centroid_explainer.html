What does "building centroids" mean?

A visual walkthrough of how the embedding router in llm/embedding_router.py turns a handful of example sentences into three reference points — and how it uses them to classify new queries without calling an LLM.

1Every sentence becomes a point

An embedding model converts each example query into a list of numbers (a vector) that captures its meaning. We can't draw a 1536-dimensional space, so imagine squishing it down to 2D: sentences with similar meaning land near each other.

small — greetings, quick facts frontier — complex reasoning specialist — support / billing

Each dot = one example sentence from prompts/route_examples.py, placed by what it means. Sentences about the same kind of thing naturally cluster together.

2The centroid is just the average position of a cluster

For each tier, we add up the coordinates of all its example points and divide by the count — exactly like averaging a list of numbers, just done in many dimensions at once. The result is one new point sitting at the "center of mass" of that cluster: the centroid.

small centroid frontier centroid specialist centroid

Click the button — each ring converges to the average of its cluster's dots. That's _build_centroids(): flat_vectors[offset:offset+count].mean(axis=0).

This only happens once (cached by get_embedding_router() as a singleton) — not on every request. After this step we throw away the 18 individual example points and keep just the 3 centroids.

3Classifying a new query: closest centroid wins

When a real query comes in, we embed it into the same space and measure how close it is to each of the three centroids (cosine similarity — a score from -1 to 1, where higher means "points in the same direction / similar meaning"). Try the examples below:

Why bother averaging instead of comparing to all 18 examples?

Without centroids

Every incoming query would need 18 similarity comparisons (one per example sentence), then some way to aggregate "which group won."

With centroids

Pre-average each group of 6 down to 1 reference point. Now classification is just 3 comparisons — fast, simple, and the noisy individual examples get smoothed into one stable "average meaning."

Where this lives in the code

In llm/embedding_router.py:

  • _build_centroids() — embeds all 18 reference sentences in one batch call, slices them back into their tiers, and averages each slice into a centroid (steps 1 & 2 above).
  • classify() — embeds the incoming query, computes cosine similarity to each of the 3 centroids, and picks the winner only if it's confident enough and clearly ahead of the runner-up (step 3 above). Otherwise it returns None and route_query in nodes/nodes.py falls back to the LLM router.
